## Lily Chat Server 生产环境配置说明

本说明用于在正式环境部署 `lily-chat-server` 时快速检查环境变量和关键配置，建议在每次上线前对照一遍。

### 1. 必须配置的环境变量

复制 `env.example` 为 `.env` 后，至少需要根据实际情况修改以下变量：

- **数据库**
  - `MONGODB_URI`：指向生产 MongoDB 的连接串，例如  
    `mongodb://127.0.0.1:27017/lily-chat`

- **鉴权与加密**
  - `JWT_SECRET`：JWT 签名密钥，**必须是强随机值，至少 32 字节**。  
    推荐在服务器上生成：  
    ```bash
    node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
    ```
  - `JWT_EXPIRES_IN`：访问 token 过期时间，例如 `7d`
  - `TOKEN_REFRESH_EXPIRES_IN`：刷新 token 过期时间，例如 `30d`
  - `ENCRYPTION_KEY`：用于加密身份证号等敏感信息，**同样必须是强随机值，至少 32 字节**。  
    生成方式同上。

- **服务器基本信息**
  - `PORT`：后端监听端口，生产默认 `8083`，需与 Nginx 反向代理保持一致。
  - `NODE_ENV`：生产环境必须设置为 `production`。

- **上传与日志**
  - `UPLOAD_PATH`：上传文件保存目录，默认 `./uploads`，需保证目录存在且 Node 进程有写权限。
  - `MAX_FILE_SIZE`：单个上传文件大小上限（字节），默认 `10485760`（约 10MB）。
  - `LOG_LEVEL`：日志级别，生产环境建议使用 `info`。

- **用户反馈（可选）**
  - `FEEDBACK_ADMIN_KEY`：设置后可通过 `GET /api/feedback` 拉取反馈列表；请求头需携带 `x-admin-key: <与变量相同的值>`。未设置或长度不足 8 时，该接口返回 404。密钥须为强随机字符串，勿提交到仓库。
  - **反馈邮件（可选）**：需同时配置 `SMTP_HOST`、`SMTP_USER`、`SMTP_PASS`（QQ 邮箱为「设置 → 账户 → 开启 SMTP」后生成的**授权码**，不是 QQ 登录密码）。`FEEDBACK_NOTIFY_EMAIL` 填管理员收件邮箱，多个用英文逗号分隔。配置正确后，用户每次提交反馈会向该邮箱发送通知。若希望向提交者发确认信，另设 `FEEDBACK_CONFIRM_EMAIL=true`（且用户个人资料里填写了邮箱）。

### 2. 建议的系统级设置

- **文件目录权限**
  - `uploads/` 和 `logs/` 目录应归属于运行 Node 进程的用户，并具有读写权限。

- **防火墙与端口**
  - 服务器应打开 `8083`（或自定义的 API 端口）对本机 / 内网可访问。
  - 对外只暴露 Nginx 的 80/443 端口，避免直接暴露 Node 端口。

### 3. 安全注意事项

- 不要在仓库中提交 `.env` 文件或任何真实密钥。
- 不要在日志中打印完整的 token、身份证号、密码等敏感信息（仅可打印掩码后的部分，用于排查）。
- 如需多人协作，建议通过运维文档和密码管理工具（如密码箱）传递环境变量，而不是聊天工具明文发散。

### 4. 上线前检查清单（生产环境）

1. [ ] `.env` 文件存在且设置了 `MONGODB_URI`, `JWT_SECRET`, `ENCRYPTION_KEY`, `PORT`, `NODE_ENV`。
2. [ ] `JWT_SECRET` 和 `ENCRYPTION_KEY` 为随机生成的长字符串，而非默认占位值。
3. [ ] MongoDB 实例可正常连接，数据库名称为 `lily-chat` 或你约定的生产库名。
4. [ ] Nginx 配置中 API 反向代理的目标端口与 `PORT` 保持一致（默认 8083）。
5. [ ] `uploads/` 和 `logs/` 目录存在且可写。
